# 财报 PDF 下载工具开发文档

## 1. 项目目标

开发一个本地运行的财报 PDF 下载工具，用于通过 AKShare/巨潮资讯数据源批量检索并下载 A 股、港股上市公司的年报、半年报、一季报、三季报等公告 PDF。

目标体验参考截图：深色桌面工具风格，左侧为查询与批量任务配置，右侧为下载日志与任务结果，底部显示全局状态。工具优先服务“投研资料批量归档”场景，而不是行情看板。

## 2. 可行性评估

### 2.1 AKShare 能力

AKShare 官方股票数据文档提供 `stock_zh_a_disclosure_report_cninfo` 接口，可查询巨潮资讯公告。该接口参数包括：

| 参数 | 说明 |
| --- | --- |
| `symbol` | 股票代码，如 A 股 `000001`，港股 `00700` |
| `market` | 市场，支持 `沪深京`、`港股`、`三板`、`基金`、`债券` 等 |
| `keyword` | 搜索关键词 |
| `category` | 公告类别，如 `年报`、`半年报`、`一季报`、`三季报` |
| `start_date` / `end_date` | 查询日期范围，格式 `YYYYMMDD` |

接口返回字段包括 `代码`、`简称`、`公告标题`、`公告时间`、`公告链接`。

官方参考：

- AKShare GitHub: https://github.com/akfamily/akshare
- AKShare 股票数据文档: https://akshare.akfamily.xyz/data/stock/stock.html

### 2.2 直接下载 PDF 的关键点

AKShare 公共返回值中的 `公告链接` 通常是巨潮详情页链接，不一定直接是 PDF 链接。但巨潮底层接口返回 `adjunctUrl` 字段，可拼接为直接 PDF 地址：

```text
http://static.cninfo.com.cn/{adjunctUrl}
```

已验证示例：

```text
000001 平安银行 2024年年度报告
adjunctUrl: finalpage/2025-03-15/1222806505.PDF
PDF: http://static.cninfo.com.cn/finalpage/2025-03-15/1222806505.PDF
```

因此实现上建议：

1. 使用 AKShare 或同源巨潮接口完成公告检索。
2. 保留 `announcementId`、`orgId`、`adjunctUrl` 等底层字段。
3. 下载时优先使用 `static.cninfo.com.cn/{adjunctUrl}`。
4. 若只能拿到 AKShare 的 `公告链接`，则从详情页或 `announcementId` 二次解析 PDF。

### 2.3 A 股支持结论

A 股可行性高。巨潮资讯对 A 股公告覆盖完整，年报、半年报、一季报、三季报可通过 `category` 精准筛选。

推荐映射：

| UI 报告类型 | AKShare/巨潮类别 | 关键词兜底 |
| --- | --- | --- |
| 年报 | `年报` | `年度报告` |
| 一季报 | `一季报` | `第一季度报告` |
| 半年报 | `半年报` | `半年度报告` |
| 三季报 | `三季报` | `第三季度报告` |

### 2.4 港股支持结论

港股可行，但需要比 A 股更谨慎：

- 港股代码需要 5 位格式，如腾讯控股为 `00700`，阿里巴巴-W 为 `09988`。
- 巨潮 `hke_stock.json` 可以提供港股代码与 `orgId` 映射。
- 港股公告用 A 股 `category=年报` 可能查不到结果，实测 `00700` 用 `keyword=年报` 可查到年报，用 `keyword=中期报告` 可查到中报。
- 港股公司并非都披露“一季报/三季报”，具体取决于上市地规则与公司披露习惯。

推荐港股匹配策略：

| UI 报告类型 | 主策略 | 兜底策略 |
| --- | --- | --- |
| 年报 | `keyword=年报` | `keyword=年度报告` |
| 半年报 | `keyword=中期报告` | `keyword=半年报`、`keyword=中报` |
| 一季报 | `keyword=第一季度` | `keyword=一季报`、`keyword=季度报告` |
| 三季报 | `keyword=第三季度` | `keyword=三季报`、`keyword=季度报告` |

## 3. 推荐技术方案

### 3.1 总体架构

推荐使用“Python 后端 + Web 前端”的本地应用架构：

```text
React/Vite UI
    |
    | HTTP + Server-Sent Events 或 WebSocket
    v
FastAPI Backend
    |
    | AKShare + requests
    v
CNInfo / AKShare Data Source
    |
    v
Local PDF Files + SQLite Task DB
```

### 3.2 为什么不直接用纯 Streamlit

当前本机已安装 Streamlit，MVP 可以很快做出可用版本。但考虑批量下载、任务队列、实时日志、暂停/重试、失败列表、后续桌面打包等需求，Streamlit 会偏吃力。

建议：

- 快速验证版本：Streamlit。
- 正式版本：FastAPI + React/Vite。
- 需要双击启动时：后续用 Tauri/Electron 或 PyInstaller 包装启动器。

### 3.3 后端模块划分

```text
backend/
  app/
    main.py                  # FastAPI 入口
    config.py                # 下载目录、并发数、请求超时等配置
    models.py                # Pydantic 请求/响应模型
    services/
      cninfo_client.py       # 巨潮接口访问与 PDF 地址解析
      akshare_client.py      # AKShare 包装层
      report_matcher.py      # 年份、报告类型、标题过滤
      downloader.py          # PDF 下载、断点/重试、文件命名
      task_queue.py          # 批量任务队列与状态流
      excel_importer.py      # Excel/CSV 导入股票代码
    storage/
      database.py            # SQLite 初始化
      repositories.py        # 任务、报告、日志持久化
```

### 3.4 前端模块划分

```text
frontend/
  src/
    App.tsx
    components/
      AppShell.tsx           # 类截图整体框架
      QueryPanel.tsx         # 单股/多股查询
      ReportSettings.tsx     # 年份、报告类型、市场
      SaveDirectory.tsx      # 保存目录
      BatchList.tsx          # 批量股票列表
      DownloadLog.tsx        # 实时日志
      TaskTable.tsx          # 成功/失败/跳过结果表
      StatusBar.tsx          # 底部状态
    api/
      client.ts
      events.ts
    styles/
      theme.css
```

## 4. 核心功能范围

### 4.1 MVP 必做功能

1. 单股下载
   - 输入股票代码。
   - 选择市场：A 股 / 港股。
   - 选择年份：如 `2025`。
   - 选择报告类型：年报 / 一季报 / 半年报 / 三季报。
   - 选择保存目录。
   - 点击“直接下载”。

2. 批量下载
   - 支持手动添加多个股票代码。
   - 支持导入 Excel/CSV。
   - 支持 A 股与港股混合列表，或统一选择市场。
   - 支持在界面上调整请求间隔时间，默认 `2.0 秒`，最小 `1.0 秒`。
   - 支持调整下载并发数，默认 `1`，建议最大不超过 `3`。
   - 下载结果分为成功、失败、跳过。

3. 下载日志
   - 实时展示每个股票的查询、匹配、下载、保存结果。
   - 支持清空日志。
   - 支持导出日志。

4. 文件命名
   - 默认命名格式：

```text
{market}_{code}_{name}_{year}_{report_type}_{announcement_date}.pdf
```

示例：

```text
A股_000001_平安银行_2024_年报_2025-03-15.pdf
港股_00700_腾讯控股_2024_年报_2025-04-08.pdf
```

5. 自动年份匹配
   - 用户选择 `2024 年报` 时，查询区间默认覆盖 `2025-01-01` 到 `2025-12-31`，因为年报通常在下一年披露。
   - 若指定区间找不到，自动扩大到前后一年，并在日志说明。

6. 源站保护
   - 批量任务必须经过统一限速器，不允许每个任务线程各自直接请求源站。
   - 查询公告与下载 PDF 都计入请求间隔。
   - 遇到超时、`429`、`403`、连接重置等情况时自动降低速度，而不是立即密集重试。

### 4.2 第二阶段功能

1. 年份批量匹配
   - 支持选择多个年份，如 `2020-2025`。
   - 支持“缺哪个补哪个”，避免重复下载。

2. 智能去重
   - 优先下载“年度报告”正本，跳过“摘要”“英文版”“取消”“更正前版本”。
   - 对“更正后年度报告”可提供优先级策略。

3. 任务控制
   - 暂停、继续、取消。
   - 失败任务一键重试。
   - 并发数设置，默认 1。

4. 报告索引
   - SQLite 记录已下载文件、公告标题、PDF URL、保存路径、下载时间。
   - 前端可按代码、年份、报告类型搜索本地文件。

### 4.3 暂不做或谨慎做

1. 不承诺所有港股都有一季报和三季报。
2. 不做投资分析和财务指标解读。
3. 不绕过网站访问限制、验证码或登录限制。
4. 不默认覆盖用户已有 PDF 文件。

## 5. UI 设计说明

### 5.1 视觉方向

参考截图采用黑色/深灰背景、荧光黄色强调色、紧凑工具面板。建议保留这种“本地投研工具”的工业感，但细节上做得更稳定：

- 背景：接近黑色 `#0b0c0d`。
- 面板：深灰 `#151617`。
- 边框：低亮灰 `#343638`。
- 主强调：荧光黄 `#f3ff3d`。
- 成功：绿色 `#22c55e`。
- 失败：红色 `#ef4444`。
- 警告：琥珀色 `#f59e0b`。

整体不要做营销页，不要做大 Hero。首屏就是工具本身。

### 5.2 首屏布局

```text
┌─────────────────────────────────────────────────────────────┐
│ 顶部栏：闪电图标 + 财报PDF下载工具        PDF下载 年份匹配 支持 │
├───────────────────┬─────────────────────────────────────────┤
│ 左侧操作区         │ 右侧日志/结果区                          │
│ ┌ 单股 / 多股 ┐    │ ┌ 下载日志                         清空 ┐ │
│ │ 股票代码    │    │ │ 实时日志流                            │ │
│ │ 加入批量    │    │ │                                      │ │
│ │ 直接下载    │    │ └──────────────────────────────────────┘ │
│ ├ 报告设置 ──┤    │ ┌ 结果表：成功/失败/跳过/路径          ┐ │
│ │ 年份 类型 市场 │  │ └──────────────────────────────────────┘ │
```

### 5.3 交互细节

1. 单股查询 Tab
   - 股票代码输入框支持 `000001`、`600519`、`00700`。
   - 输入港股 `700` 时自动补齐为 `00700`，但日志提示规范化结果。

2. 多股查询 Tab
   - 批量文本框支持换行、逗号、空格分隔。
   - Excel/CSV 导入支持列名：`code`、`股票代码`、`证券代码`。
   - 批量列表显示代码、市场、公司简称、状态。

3. 报告设置
   - 年份为数字输入或下拉。
   - 报告类型用 segmented control。
   - 市场用 segmented control：A 股 / 港股 / 自动识别。

4. 批量保护设置
   - 请求间隔使用 slider + 数字输入，单位为秒。
   - 默认值：`2.0 秒`。
   - 可选范围：`1.0-10.0 秒`。
   - 并发数使用 stepper，默认 `1`，范围 `1-3`。
   - UI 文案使用“请求间隔”和“并发数”，避免暗示越快越好。
   - 当失败率升高或出现限流状态码时，状态栏提示“已自动降速”。

5. 下载日志
   - 每条日志包含时间、级别、股票代码、消息。
   - 错误日志可点击查看详情。
   - 限速等待也写入轻量日志，如“等待 2.0 秒后继续请求”。

6. 结果表
   - 成功项显示本地文件路径。
   - 失败项显示失败原因。
   - 跳过项显示原因，如“文件已存在”“未找到正本报告”。

### 5.4 源站保护设置面板

建议放在左侧“报告设置”和“保存目录”之间：

```text
┌ 请求设置 ─────────────────────┐
│ 请求间隔  2.0 秒   [----●---] │
│ 并发数    1        [-] 1 [+]  │
│ □ 遇到限流自动降速             │
└───────────────────────────────┘
```

设计原则：

- 默认偏保守，用户需要明确调高速度。
- 即使用户设置并发数为 3，也要由全局限速器控制发往同一域名的请求节奏。
- 前端只传用户期望值，后端仍要校验最小间隔和最大并发。

## 7. 报告匹配规则

### 7.1 年份识别

标题中必须包含目标年份或可推导目标年份：

- `2024年年度报告` -> `2024 年报`
- `2024 第一季度报告` -> `2024 一季报`
- `中期报告 2024` -> `2024 半年报`

### 7.2 标题优先级

下载正本报告时按以下规则打分：

| 条件 | 分值 |
| --- | --- |
| 标题包含目标年份 | +40 |
| 标题包含目标报告类型关键词 | +40 |
| 标题不包含“摘要” | +20 |
| 标题包含“更正后” | +10 |
| 标题包含“英文版” | -20 |
| 标题包含“摘要” | -40 |
| 标题包含“取消”或“作废” | -100 |

取最高分候选；若最高分低于阈值，标记为未找到。

### 7.3 重复文件策略

默认不覆盖。若目标路径已存在：

1. 校验文件大小大于 0。
2. 记录为 `skipped`。
3. UI 中展示“已存在”。

可在设置中提供“覆盖已存在文件”选项。

## 9. 下载流程

```text
用户提交任务
  -> 校验保存目录
  -> 校验请求间隔和并发数
  -> 规范化股票代码
  -> 识别市场或使用用户选择
  -> 进入全局限速器
  -> 查询公告候选
  -> 根据设置等待请求间隔
  -> 按年份与报告类型打分
  -> 生成 PDF URL
  -> 再次进入全局限速器
  -> 下载到临时文件
  -> 校验 Content-Type / 文件大小 / PDF 头
  -> 原子重命名为目标文件
  -> 写入 SQLite 任务结果
  -> 推送日志与状态
```

### 9.1 限速策略

1. 全局限速
   - 对 `cninfo.com.cn` 与 `static.cninfo.com.cn` 统一限速。
   - 默认每次源站请求之间至少间隔 `2.0 秒`。
   - 用户可在 UI 调整，但后端强制最小 `1.0 秒`。

2. 并发控制
   - 默认并发数为 `1`。
   - 最大允许 `3`。
   - 即使并发数大于 1，同一域名请求也必须排队经过限速器。

3. 自动降速
   - 出现 `429`、`403`、`5xx`、连接重置、连续超时等情况时，临时把请求间隔乘以 `2`。
   - 最大退避到 `60 秒`。
   - 连续成功若干次后逐步恢复到用户设置值。

4. 重试策略
   - 单个请求最多重试 `3` 次。
   - 重试必须遵守当前退避间隔。
   - 不对同一个失败 URL 进行快速循环请求。

5. 日志透明
   - 日志中显示当前间隔、降速原因、恢复速度等信息。
   - 批量任务开始时记录本次任务的限速参数。

## 10. 错误处理

| 场景 | 前端提示 | 后端处理 |
| --- | --- | --- |
| 股票代码不存在 | 代码未找到 | 标记失败，不中断整个批次 |
| 未找到目标报告 | 未找到对应报告 | 扩大日期范围后仍无结果则失败 |
| 网络超时 | 网络超时，已重试 | 指数退避重试 3 次 |
| 源站限流 | 源站繁忙，已自动降速 | 增大请求间隔并继续队列 |
| PDF 下载失败 | PDF 下载失败 | 保留候选信息，允许重试 |
| 文件已存在 | 已存在，跳过 | 默认不覆盖 |
| 保存目录无权限 | 保存目录不可写 | 阻止任务开始 |
| 港股无季度报告 | 未找到披露 | 记录为未找到，不视作程序异常 |

## 11. 环境与依赖

## 13. 验收标准

1. 输入 `000001`、A 股、`2024`、年报，可下载平安银行 2024 年年度报告 PDF。
2. 输入 `00700`、港股、`2024`、年报，可下载腾讯控股 2024 年报 PDF。
3. 批量输入 10 个股票代码时，单个失败不影响其他任务。
4. 下载日志能实时显示进度。
5. 批量下载默认请求间隔不小于 `2.0 秒`，界面可调整，后端强制最小 `1.0 秒`。
6. 出现限流或连续超时时，任务会自动降速并在日志中说明。
7. 已存在文件默认跳过，不覆盖。
8. 保存文件名可读、稳定、无非法字符。
9. 无报告时明确显示“未找到”，而不是静默失败。

## 14. 主要风险

1. 巨潮接口结构变化：需要把数据源访问集中封装在 `cninfo_client.py`，便于替换。
2. 港股季度报告覆盖不稳定：UI 和日志必须明确说明“未披露/未找到”。
3. 批量请求过快被限流：默认低并发，提供可调请求间隔，并加入全局限速器、自动降速和退避重试。
4. 本地路径权限问题：任务开始前检查目录可写性。
5. 文件名包含非法字符：统一清洗公司简称和标题。
