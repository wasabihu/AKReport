## 1. 前端目标

前端负责提供一个本地投研工具界面，用于单股/多股选择、报告参数配置、源站保护参数配置、任务创建、实时日志展示、下载结果展示和任务控制。

前端不负责：

- 直接访问 AKShare 或巨潮资讯。
- 拼接 PDF URL。
- 判断报告匹配是否正确。
- 判断文件是否下载成功。

这些逻辑全部由后端负责。

跨平台约束：

- 前端默认支持 macOS 和 Windows 浏览器访问。
- 保存目录只作为字符串传给后端，前端不自行解析平台路径。
- UI 不得写死 `/Users/...` 或 `C:\...` 作为固定默认值。
- Windows 长路径在表格中需要可读展示，避免撑破布局。

## 6. 组件职责

### 6.1 AppShell

职责：

- 组织整体布局。
- 提供页面级 loading/error boundary。
- 注入当前任务状态。

不负责：

- API 细节。
- 任务事件解析。
- 表单字段校验细节。

### 6.2 QueryTabs

职责：

- 在“单股查询”和“多股查询”之间切换。
- 保持两个 tab 的输入状态。
- 切换时不清空已输入内容。

### 6.3 SingleQueryForm

字段：

- `code`
- `market`

行为：

- 输入 A 股 6 位代码时保持原样。
- 输入港股 1-5 位数字时 UI 可提示会由后端规范化。
- 前端只做基础格式校验，不做最终市场判定。

### 6.4 BatchQueryForm

职责：

- 支持多行、逗号、空格分隔。
- 展示解析后的代码列表。
- 支持删除单个代码。
- 支持清空列表。

MVP 文本解析：

```text
000001
600519, 00700
09988 000858
```

解析结果：

```ts
["000001", "600519", "00700", "09988", "000858"]
```

Excel/CSV 导入第二阶段实现，组件接口先预留。

### 6.5 ReportSettings

字段：

- `years`
- `reportTypes`
- `marketMode`

控件：

- 年份：数字输入，MVP 单年，第二阶段支持范围。
- 报告类型：segmented control 或 checkbox group。
- 市场：segmented control，`自动识别 / A 股 / 港股`。

### 6.6 RequestSettings

这是源站保护参数，必须在 UI 中可见。

字段：

- `requestIntervalSeconds`
- `concurrency`
- `autoSlowdown`

控件：

```text
请求间隔  2.0 秒   [----●---]
并发数    1        [-] 1 [+]
[x] 遇到限流自动降速
```

校验：

- 请求间隔范围：`1.0-10.0`。
- 步进：`0.5`。
- 并发数范围：`1-3`。
- 默认：间隔 `2.0`，并发 `1`，自动降速开启。

说明：

- 前端应阻止明显非法值。
- 后端仍会二次校验，前端不能依赖自己校验作为安全边界。

### 6.7 SaveDirectory

MVP：

- 文本输入保存目录。
- 显示当前目录。
- 后端校验是否可写。
- 输入框必须允许 Windows 路径和 macOS/POSIX 路径。
- 表格中的长路径需要省略显示，不允许撑破布局。

桌面打包后：

- 接入系统目录选择器。

### 6.8 ActionBar

按钮：

- 加入批量列表
- 直接下载

- 任务完成后允许重试失败。

### 6.9 LogPanel

## 7. TypeScript 类型

前端类型必须与后端 API 保持一致。建议先写在 `src/api/types.ts`。

```ts
export type Market = 'A股' | '港股' | 'auto';
export type ReportType = '年报' | '一季报' | '半年报' | '三季报';
export type TaskStatus = 'pending' | 'running' | 'paused' | 'completed' | 'failed' | 'cancelled';
export type ItemStatus = 'pending' | 'searching' | 'downloading' | 'success' | 'failed' | 'skipped';

export interface ReportSearchRequest {
  code: string;
  market: Market;
  year: number;
  report_type: ReportType;
}

export interface CreateTaskRequest {
  codes: string[];
  market_mode: Market;
  years: number[];
  report_types: ReportType[];
  save_dir: string;
  request_interval_seconds: number;
  concurrency: number;
  auto_slowdown: boolean;
  overwrite_existing: boolean;
}

export interface TaskItem {
  id: string;
  code: string;
  market: Market;
  year: number;
  report_type: ReportType;
  status: ItemStatus;
  message: string;
  name?: string;
  file_path?: string;
  announcement_title?: string;
  pdf_url?: string;
}

export interface TaskDetail {
  id: string;
  status: TaskStatus;
  items: TaskItem[];
  stats: {
    total: number;
    success: number;
    failed: number;
    skipped: number;
    pending: number;
  };
}

export interface TaskLogEvent {
  time: string;
  level: 'debug' | 'info' | 'warn' | 'error';
  task_id: string;
  code?: string;
  message: string;
}
```

注意：API JSON 字段统一使用 snake_case，前端不要擅自改成 camelCase，除非在 API client 做集中转换。

## 8. API Client

### 8.1 基础封装

注意：当 `init.body` 为 `FormData` 时，自动跳过 `Content-Type` header，避免破坏 multipart 边界。

```ts
export class ApiError extends Error {
  constructor(
    message: string,
    public code?: string,
    public status?: number,
  ) {
    super(message);
  }
}

export async function apiFetch<T>(path: string, init?: RequestInit): Promise<T> {
  const response = await fetch(`/api${path}`, {
    headers: { 'Content-Type': 'application/json', ...(init?.headers ?? {}) },
    ...init,
  });

  if (!response.ok) {
    const payload = await response.json().catch(() => null);
    throw new ApiError(
      payload?.error?.message ?? `请求失败：${response.status}`,
      payload?.error?.code,
      response.status,
    );
  }

  return response.json() as Promise<T>;
}
```

### 8.2 端点函数

```ts
export function createTask(input: CreateTaskRequest) {
  return apiFetch<{ data: { task_id: string; status: TaskStatus }; message: string }>('/tasks', {
    method: 'POST',
    body: JSON.stringify(input),
  });
}

export function getTask(taskId: string) {
  return apiFetch<{ data: TaskDetail; message: string }>(`/tasks/${taskId}`);
}

export function cancelTask(taskId: string) {
  return apiFetch<{ data: TaskDetail; message: string }>(`/tasks/${taskId}/cancel`, {
    method: 'POST',
  });
}
```

## 9. SSE 事件处理

### 9.1 连接

```ts
export function subscribeTaskEvents(taskId: string, handlers: TaskEventHandlers): EventSource {
  const source = new EventSource(`/api/tasks/${taskId}/events`);

  source.addEventListener('log', (event) => {
    handlers.onLog(JSON.parse((event as MessageEvent).data));
  });

  source.addEventListener('item_updated', (event) => {
    handlers.onItemUpdated(JSON.parse((event as MessageEvent).data));
  });

  source.addEventListener('task_completed', (event) => {
    handlers.onTaskCompleted(JSON.parse((event as MessageEvent).data));
    source.close();
  });

  source.onerror = () => {
    handlers.onError?.();
  };

  return source;
}
```

### 9.2 断线恢复

MVP 要求：

- SSE 断开时显示“日志连接断开，正在重连”。
- 重连前调用 `GET /api/tasks/{task_id}` 拉取当前任务状态。
- 避免重复追加完全相同的日志，可按日志 ID 或 `time+message` 简单去重。

## 10. 状态管理

MVP 可用 React state + reducer，不必引入复杂状态库。

```ts
interface TaskState {
  currentTaskId?: string;
  status: TaskStatus | 'idle';
  items: TaskItem[];
  logs: TaskLogEvent[];
  stats: TaskDetail['stats'];
  rateLimitNotice?: string;
}
```

Reducer action：

```ts
type TaskAction =
  | { type: 'task_created'; taskId: string }
  | { type: 'task_loaded'; task: TaskDetail }
  | { type: 'log_received'; log: TaskLogEvent }
  | { type: 'item_updated'; item: Partial<TaskItem> & { id?: string; code?: string } }
  | { type: 'task_completed'; status: TaskStatus }
  | { type: 'reset' };
```

状态原则：

- 后端是任务状态事实来源。
- 前端乐观更新只用于按钮 loading，不用于最终结果。
- 创建新任务前，明确清空或归档上一任务的 UI 状态。

## 11. 表单校验

### 11.1 股票代码

前端基础规则：

- 只允许数字、空格、逗号、换行。
- A 股通常 6 位。
- 港股允许 1-5 位，提示后端会补零。
- 自动模式不强制市场。

### 11.2 年份

- MVP 单年，范围建议 `1990` 到当前年份。
- 后续支持范围，如 `2020-2025`。

### 11.3 请求设置

- `request_interval_seconds` 必须在 `1.0-10.0`。
- `concurrency` 必须在 `1-3`。
- 自动降速默认开启。

非法输入必须在创建任务前阻止，并显示紧邻控件的错误。

## 14. 可访问性和键盘操作

MVP 要求：

- 所有按钮可通过键盘聚焦。
- 输入框有 label 或 `aria-label`。
- segmented control 使用 button，而不是不可聚焦 div。
- 状态色不能作为唯一信息，必须有文字。
- 日志区域使用 `aria-live="polite"`，但避免频繁朗读造成干扰。

## 17. 前后端协作契约

前端开工前需要后端确认：

- API base path 为 `/api`。
- JSON 字段保持 snake_case。

- `src/api/types.ts`
