# 后端开发文档

## 1. 后端目标

后端负责财报公告检索、报告匹配、PDF 下载、批量任务编排、源站限速保护、文件落盘、任务日志和本地索引。前端不得绕过后端直接访问巨潮或拼接 PDF URL。

核心原则：

1. 所有源站请求必须经过全局限速器。
2. 所有功能改动必须测试优先。
3. 批量任务中单个失败不能中断整个任务。
4. 下载文件必须可校验、可追踪、不可误覆盖。
5. 默认支持 macOS 和 Windows，本地路径必须使用 `pathlib.Path`，不得写死平台专属路径。

## 2. 技术栈

| 类别 | 选择 | 说明 |
| --- | --- | --- |
| Python | `3.11+` | 避免使用全局 Python 环境 |
| Web 框架 | `FastAPI` | 提供 REST API 和 SSE |
| 数据模型 | `Pydantic v2` | 入参、出参、配置校验 |
| HTTP 客户端 | `httpx` | 支持 timeout、重试包装、异步扩展 |
| 数据处理 | `pandas`、`openpyxl` | AKShare 和 Excel 导入 |
| 数据源 | `akshare`、巨潮公开接口 | AKShare 为主，必要时访问巨潮底层接口 |
| 存储 | `SQLite` | 本地任务和下载索引 |
| 测试 | `pytest`、`respx` 或 `pytest-httpx` | Mock HTTP，减少真实源站请求 |

建议独立环境：

```bash
python3.11 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install fastapi uvicorn pydantic pydantic-settings httpx akshare pandas openpyxl pytest pytest-asyncio pytest-httpx
```

模块边界：

- `services/cninfo_client.py` 只负责巨潮请求、字段解析、PDF URL 解析。
- `services/akshare_client.py` 只负责 AKShare 包装和兼容性隔离。
- `services/report_matcher.py` 只负责候选报告评分和选择。
- `services/rate_limiter.py` 只负责限速、退避、恢复。
- `services/downloader.py` 只负责下载、校验、临时文件、原子重命名。
- `services/task_queue.py` 只负责任务状态机和批量调度。
- `storage/` 只负责 SQLite 读写。

## 4. 配置设计

使用 `pydantic-settings`：

```python
class Settings(BaseSettings):
    app_name: str = "AKShare Wasa"
    database_path: Path = Path("data/app.sqlite3")
    default_save_dir: Path = Field(default_factory=default_save_dir)
    default_request_interval_seconds: float = 2.0
    min_request_interval_seconds: float = 1.0
    max_concurrency: int = 3
    request_timeout_seconds: float = 20.0
    max_retries: int = 3
    max_backoff_seconds: float = 60.0
    cninfo_base_url: str = "http://www.cninfo.com.cn"
    cninfo_static_base_url: str = "http://static.cninfo.com.cn"
```

约束：

- 用户传入 `request_interval_seconds < min_request_interval_seconds` 时返回 `400`。
- 用户传入 `concurrency > max_concurrency` 时返回 `400`。
- 保存目录必须在任务开始前检查可写。
- 默认保存目录：Windows 使用 `C:/reports`，macOS/Linux 使用 `Path.home() / "Downloads" / "reports"`。

## 8. 巨潮/AKShare 封装

### 8.1 股票字典

巨潮股票字典：

```text
A 股:  http://www.cninfo.com.cn/new/data/szse_stock.json
港股: http://www.cninfo.com.cn/new/data/hke_stock.json
```

封装要求：

- 缓存股票字典，默认进程内缓存。
- 缓存失效时间建议 24 小时。
- 缓存刷新也必须走限速器。
- 港股代码查找前必须补齐 5 位。

### 8.2 公告查询

巨潮查询接口：

```text
POST http://www.cninfo.com.cn/new/hisAnnouncement/query
```

核心 payload：

```python
{
    "pageNum": "1",
    "pageSize": "30",
    "column": "szse" or "hke",
    "tabName": "fulltext",
    "stock": f"{code},{org_id}",
    "searchkey": keyword,
    "category": category,
    "seDate": "2025-01-01~2025-12-31",
    "isHLtitle": "true",
}
```

实现要求：

- 分页拉取所有候选。
- 保留 `announcementId`、`orgId`、`adjunctUrl`、`announcementTitle`、`announcementTime`。
- `adjunctUrl` 拼接为：

```text
http://static.cninfo.com.cn/{adjunctUrl}
```

### 8.3 日期范围推导

默认规则：

| 报告类型 | 用户选 2024 时默认查询区间 |
| --- | --- |
| 年报 | `2025-01-01~2025-12-31` |
| 一季报 | `2024-03-01~2024-06-30` |
| 半年报 | `2024-07-01~2024-10-31` |
| 三季报 | `2024-09-01~2024-12-31` |

兜底规则：

- 默认区间无结果时，扩展到前后一年。
- 扩展查询必须写日志。
- 扩展查询仍要遵守限速器。

## 10. 限速器设计

### 10.1 接口

```python
class RateLimiter:
    async def acquire(self, domain: str) -> None: ...
    async def record_success(self, domain: str) -> None: ...
    async def record_failure(self, domain: str, reason: str) -> None: ...
    def snapshot(self) -> RateLimitSnapshot: ...
```

### 10.2 行为

- 每个域名维护 `last_request_at`、`current_interval`、`failure_count`。
- 请求前调用 `acquire(domain)`。
- `acquire` 根据 `last_request_at + current_interval` 等待。
- 失败时 `current_interval = min(current_interval * 2, max_backoff_seconds)`。
- 连续成功后逐步恢复到用户设置的基础间隔。

### 10.3 测试要求

先写以下测试：

- 默认间隔为 `2.0 秒`。
- 用户间隔低于 `1.0 秒` 时配置校验失败。
- 同一域名连续请求会等待。
- 不同域名也应可配置为统一限速或独立限速，MVP 统一处理巨潮两个域名。
- `429` 后间隔翻倍。
- 连续成功后间隔恢复。

## 11. 下载器设计

流程：

```text
生成目标文件名
  -> 检查是否存在
  -> 存在且不覆盖则 skipped
  -> 下载到 .partial
  -> 校验状态码、大小、PDF 文件头
  -> 原子重命名
  -> 写入结果
```

文件名清洗：

- 替换 `/\:*?"<>|` 为 `_`。
- 去掉连续空白。
- 文件名长度建议限制在 180 字符内。
- 路径拼接必须使用 `pathlib.Path`，不得用字符串拼接 `/` 或 `\`。
- 测试必须覆盖 Windows 风格路径和 macOS/POSIX 风格路径。

下载校验：

- `Content-Type` 允许 `application/pdf`、`application/octet-stream` 或为空。
- 文件头优先检查 `%PDF`。
- 文件大小为 0 直接失败。

## 12. 任务队列

### 12.1 状态机

```text
pending -> running -> completed
pending -> running -> failed
pending -> running -> cancelled
running -> paused -> running
```

### 12.2 Item 流程

```text
pending
  -> searching
  -> downloading
  -> success / failed / skipped
```

### 12.3 并发策略

- MVP 使用 `asyncio.Queue`。
- worker 数量等于用户设置的 `concurrency`。
- 默认 worker 数为 `1`。
- 每个 worker 中所有源站请求都必须经过同一个 `RateLimiter` 实例。
- 任务取消时停止取新 item，正在下载的 item 尽量安全结束。

## 14. 错误码

| code | HTTP | 说明 |
| --- | --- | --- |
| `invalid_stock_code` | 400 | 股票代码格式错误 |
| `invalid_rate_limit` | 400 | 请求间隔或并发数非法 |
| `save_dir_not_writable` | 400 | 保存目录不可写 |
| `task_not_found` | 404 | 任务不存在 |
| `report_not_found` | 200/item failed | 未找到对应报告 |
| `source_rate_limited` | 200/item retrying | 源站限流，已降速 |
| `download_failed` | 200/item failed | PDF 下载失败 |
| `internal_error` | 500 | 未预期错误 |

## 15. 测试优先清单

开发前必须先写测试：

1. `normalize_stock_code("700", 港股) -> "00700"`。
2. `normalize_stock_code("000001", A股) -> "000001"`。
3. 年报标题评分高于摘要标题。
4. “取消”“作废”标题得分低于阈值。
5. 默认查询 2024 年报时使用 2025 年披露区间。
6. 限速器拒绝低于 1 秒的间隔。
7. 限速器遇到 429 自动翻倍。
8. 下载器遇到已有文件返回 skipped。
9. 下载器遇到非 PDF 文件返回 failed。
10. 年报候选小于 1MB 时被匹配器过滤。
11. 年报实际下载后小于 1MB 时返回 failed 且不落盘。
12. 小于 1MB 的季报、中报不受年报阈值影响。
13. 常用股票写入会用源站字典规范化 `700 -> 00700` 并补全 `腾讯控股`。
14. 常用股票历史存在规范化冲突时会合并记录并累加使用次数。
15. 批量任务单个失败不影响其他 item。

真实源站测试只放在手动或低频集成测试中，不能作为每次单元测试默认执行。
